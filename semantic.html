Inline tags

The <strong> element is used to define important text, but <b> is only used for visual styles.

The <em> element is used to define emphasized text, but <i> is only used for visual styles.

The <mark> element is used to highlight text.

The <del> element is used to define deleted text.

The <ins> element is used to define inserted text.

The <sub> element is used to define subscripted text.

The <sup> element is used to define superscripted text.

The <q> element is used because different languages have different quotation marks. This element handles that for you.
What's in a name?, says Juliet. Use the <q> element instead of " or '.

The <kbd> element, used above, is used when we want to show keyboard input. Ctrl + C is the copy command.

Other inline elements include the <time>, <code>, <var>, <samp>, <dfn>, <span>, and <br> elements.

Languages

The <html> element has a lang attribute that can be used to specify the language of the document. This is important for screen readers and other accessibility tools.

Say for example you are speaking English and then suddenly vous parlez français. The lang attribute is used to specify the language of the text. It's not visually displayed, but accessibility tools can use it.

The <html> element can also have a dir attribute that can be used to specify the direction of the text. This is important for languages that are read from right to left.

Here's some Hebrew text:

נח אוהב את יעקב
Noah loves Jacob

The dir attribute is used to specify the direction of the text. Note we also used the <small> element to make the translation smaller.

Code

The <code> element is used to define a piece of computer code. The content inside is displayed in a monospace font.

The <pre> element is used to define preformatted text. The text inside a <pre> element is displayed in a fixed-width font, and it preserves both spaces and line breaks. Both <code> and <pre> are used in conjunction with each other. 

  <html> <head> <title>Title</title> </head> <body> <h1>Heading</h1> <p>Paragraph</p> </body> </html>

Tables

Table Caption. The <caption> tag is used, again, for accessibility tools to call out the name of the table/picture.
Tag name Purpose
<table> Defines a table
<tr> Defines a row in a table
<th> Defines a header cell in a table
<td> Defines a cell in a table
<thead> Groups the header content in a table. This is used for styling purposes. Also, if you are using a library like sorttable, it will ignore the header row, and add arrows to the header row.
<tbody> Groups the body content in a table. This is used for styling purposes. Also, if you are using a library like sorttable, it will only sort the tbody.

If you are using a library like sorttable, it will ignore the footer row. You can place the total of a column in the footer by using dummy <tr> tags with the colspan attribute.

Forms

There are many form elements in HTML. The <form> element is used to create an HTML form for user input. The <input> element is the most used form element. The <label> element is used to define a label for many form elements.

The <fieldset> element is used to group related elements in a form. PicoCSS allows you to also horizontally group related inputs with the role='group' on the <fieldset> element. One could also use class="grid" to achieve similar effects.

The <legend> element defines a caption for the <fieldset> element. It helps screen readers understand the purpose of the group. (Selection, Contact Information, etc.)

Images

The <img> element is used to embed an image in an HTML page. The alt attribute provides an alternate text for an image, if the image cannot be displayed.

The <img srcset> attribute

The <img srcset> attribute is used to provide multiple image files for different device resolutions. The browser will choose the most suitable image. 

  <img src="image.jpg" alt="Image" srcset="image.jpg 200w, image-2x.jpg 400w">
OR, if using pixel density descriptors: 
  <img src="image.jpg" alt="Image" srcset="image.jpg 1x, image-2x.jpg 2x">

Pixel density descriptors are used to provide images for different pixel densities. The 1x descriptor is used for regular displays, and the 2x descriptor is used for high-resolution displays.

It is incorrect to mix width descriptors and pixel density descriptors in the same srcset attribute. Duplicate descriptors (for instance, two sources in the same srcset which are both described with 2x) are also invalid.

The <img sizes> attribute

The <sizes> attribute is used to provide the browser with information about the viewport. The browser will choose the most suitable image based on the viewport.

If the srcset attribute uses width descriptors, the sizes attribute must also be present, or the srcset itself will be ignored. 

  <img src="image.jpg" alt="Image" sizes="(max-width: 600px) 200px, 50vw">

The <picture> element

The <picture> element is used to define an image. The <source> element is used to provide multiple sources for the browser to choose from. 

 <picture> 
    <source srcset="image.jpg" media="(max-width: 600px)">
    <img src="image.jpg" alt="Image"> 
</picture>

Why use a picture element?

In HTML, the "picture" element is used to provide multiple image options based on different screen sizes or conditions, while the "img" element is used to display a single image without any built-in adaptability; essentially, "picture" allows for responsive image selection, whereas "img" is a simple image embedder.

Audio

The <audio> element is used to embed sound content in an HTML document. The controls attribute adds audio controls, like play, pause, and volume.

The <source> element

The <source> element performs different functions depending on the context in which it is used. In the context of the <audio> element, the <source> element is used to specify alternative audio files which the browser may choose from. The browser will use the first recognized format. The type attribute is used to specify the media type of the src attribute. 

  <audio controls>
    <source src="horse.mp3" type="audio/mpeg">
    <source src="horse.ogg" type="audio/ogg">
    Your browser does not support the audio element.
</audio>

The <track> element

The <track> element is used to specify text tracks for video elements. The <track> element is used in conjunction with the <audio> and <video> elements. The src attribute specifies the URL of the track file. The srclang attribute specifies the language of the track text data. The label attribute specifies the title of the text track. The kind attribute specifies the kind of text track. 

  <audio controls>
    <source src="horse.mp3" type="audio/mpeg">
    <track src="subtitles_en.vtt" kind="subtitles" srclang="en" label="English">
    <track src="subtitles_no.vtt" kind="subtitles" srclang="no" label="Norwegian">
    Your browser does not support the audio element.
</audio>

Audio Attributes

The autoplay attribute is used to specify that the audio will start playing as soon as it is ready. The loop attribute is used to specify that the audio will start over again, every time it is finished. The preload attribute is used to specify that the audio will be loaded when the page loads. 

  <audio controls autoplay loop preload="auto">
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Audio Events

The onplay event occurs when the audio has started playing. The onpause event occurs when the audio has been paused. The onended event occurs when the audio has ended. 

  <audio controls onplay="alert('The audio has started playing.')" onpause="alert('The audio has been paused.')" onended="alert('The audio has ended.')">
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

Audio Methods

The play() method starts playing the audio. The pause() method pauses the audio. The load() method reloads the audio. 

  <audio id="myAudio" controls>
    <source src="horse.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

<script>
    var x = document.getElementById("myAudio");

    function playAudio() {
        x.play();
    }

    function pauseAudio() {
        x.pause();
    }
</script>

ARIA

ARIA stands for Accessible Rich Internet Applications. It is a set of attributes that define ways to make web content and web applications more accessible to people with disabilities. ARIA is used to improve the accessibility of web content for people with disabilities.

ARIA Roles

The role attribute is used to define the role of an element. The role attribute is used to define the purpose, structure, and state of an element.

ARIA States and Properties

  • The aria-checked attribute is used to define the checked state of an element. This lets screen readers know that the element is checked.
  • The aria-disabled attribute is used to define the disabled state of an element. It lets the user know that the element is disabled.
  • The aria-hidden attribute is used to define the hidden state of an element. It is used to hide an element from the screen reader.
  • The aria-invalid attribute is used to define the invalid state of an element. It can be used in forms to indicate that a field is invalid. PicoCSS uses this attribute to show a red border around the input field.
  • The aria-pressed attribute is used to define the pressed state of an element. It is used to indicate that a button is pressed.
  • The aria-selected attribute is used to define the selected state of an element. It is used to indicate that an element is selected.

ARIA Landmarks

We used the <aside> element to create a navigation menu. You can check the source code of this page to see how we used the role attribute to define the role of the <aside> element.

There are other semantic elements like the <header>, <footer>, <nav>, <main>, <article>, and <section> elements that can be used to define the structure of a web page.

They are called ARIA landmarks because they help to define the structure of a web page. They are used to define the main regions of a web page, like the header, footer, navigation, main content, and sidebars.
